    Gatsbyブログ始め方

    パソコンでやってね

    プログラミング的な準備

    1. 以下4つのインストールが必要。

    Node.js 　　　 プログラミングツール的な
    Git 　　　　　 ファイル変更履歴管理ツール
    Gatsby CLI 　　 Gatsby用ツール
    VSCode 　　　 コード編集ソフト

    インストール方法の説明はちょっとだるいわ。リンク貼るけん見て →インストール方法
    詰まったらドンマイ。ゲームオーバー。

    1. 以下コマンドを実行
    gatsby new my-blog-starter https://github.com/gatsbyjs/gatsby-starter-blog

    わからん人は「パソコン コマンド 実行」で調べてくれ。my-blog-starterの部分はパソコンにできるプロジェクト用のフォルダの名前になる（まんまでもいいしあとで変えれる）。以下my-blog-starterは自分でつけた名前に読み替えてネ

    ちなみに何をしてるかというとギャッツビーさんが用意してくれてるブログスターターをダウンロードしてる。

    1. 以下コマンドを実行
    cd my-blog-starter/
gatsby develop

    http://localhost:8000 にアクセスするとブログが表示される

    記事書く

    VSCodeでプロジェクトフォルダを開く。/content/blogフォルダの中を見るとなんとなく記事の追加の仕方がわかる。わかったらファイルを追加して記事を書く。

    http://localhost:8000 にアクセスすると記事が追加されている。はず。

    追加できたならばファイルを追加したことをGit（ファイル変更履歴管理ツール）に反映する必要がある。さっき実行したコマンドで実行画面が変なことなってると思うので、別タブを立ち上げて以下コマンドを実行。

    cd my-blog-starter
git add .
git commit -m '記事追加したよん'

    ネット上に公開

    http://localhost:8000 は自分のパソコンからだけアクセスできるインターネット、的なやつ。友達に見せれるようにするにはネット上に公開する必要がある。そのためにまずプロジェクトファイルの中身をGitHubなるサイト（登録してネ）にアップロードする必要がある。

    1. GitHubでリポジトリ（保管場所をカッコつけていった表現）を新規作成

    名前だけプロジェクトフォルダ名に合わせてつけたら他は無視。作成をクリック

    1. 以下コマンドを実行（1行目は読み替えてネ）
    git remote add origin https://github.com/YOUR_GITHUB_USERNAME/YOUR_GITHUB_REPO_NAME.git
git branch -M main
git push -u origin main

    これでパソコン上のフォルダとGitHubのリポジトリを紐付けてファイルをアップロードできた。はず。GitHubのリポジトリのページをリロードしてみてプロジェクトフォルダがアップロードされてるっぽかったらオッケー。

    1. ネットに公開

    上の手順はコードをネット上に置いただけのイメージ。それをいい感じに公開する必要がある。

    Gatsbyのサイトいわく、Gatsby Cloudなるサービスを使うのが一番簡単にネットに公開できるらしい。

    が、あえて俺は別の道をゆく。GitHub Pagesを使う。多分Gatsbyで作ったサイトを簡単に公開できるのがGatsby Cloudで、Gatsby以外で作ったサイトをこれまた別のサービスで公開。。とかするとなんか色々散らばってしまう。ので僕は何でサイト作ろうとGitHub Pagesで公開するようにしている。

    「いや、うち公開したいんこのブログくらいやし。。」て人はさっき貼ったGatsbyのサイトの説明見ながらやるんがいいと思う。ひねくれた人はこれから書くGitHub Pagesで公開する方法でやろう。

    GitHub Pagesで公開

    1. いろいろ設定

    プロジェクトフォルダ内に/.github/workflows/deploy.ymlというファイルを用意する。中身は以下をコピペ

    name: GitHub Pages

on:
  push:
    branches:
      - main
  pull_request:

jobs:
  deploy:
    runs-on: ubuntu-20.04
    concurrency:
      group: ${{ github.workflow }}-${{ github.ref }}
    steps:
      - uses: actions/checkout@v2

      - name: Setup Node
        uses: actions/setup-node@v2
        with:
          node-version: '14'

      - name: Cache dependencies
        uses: actions/cache@v2
        with:
          path: ~/.npm
          key: ${{ runner.os }}-node-${{ hashFiles('**/package-lock.json') }}
          restore-keys: |
            ${{ runner.os }}-node-

      - run: npm ci
      - run: npm run build

      - name: Deploy
        uses: peaceiris/actions-gh-pages@v3
        if: ${{ github.ref == 'refs/heads/main' }}
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./public

    ちなみに上のコードは「GitHub Pages用の設定はこー書いたらええで」ってまとめてくれてる人がおってその人のGatsbyの場合はこーやでの部分を使ってる。

    gatsby-config.jsの頭に以下1行を追加

    module.exports = {
  pathPrefix: `/my-blog-starter`, ←この1行を追加
  siteMetadata: {

    package.jsonの該当部分を以下のように変更

    - "build": "gatsby build",
+ "build": "gatsby build --prefix-paths",
    1. コミットしてプッシュ（変更履歴管理対象に加え、GitHubにアップロード）
    git add .
git commit -m '設定追加したお'
git push -u origin main
    1. GitHub上で設定

    ブログ用に作成したGitHubのリポジトリのページを開く。Settingを押す。

    左のタブの下の方のPagesってとこを押す。Sourceを選ぶとこで「gh-pages」を選ぶ(プッシュしてから2分くらい待たんと出てないかも)。Saveを押す。

    「ここに公開されます！」みたいなのが上らへんに表示されるからそのリンクを開く。2分くらい待つとブログが公開されてる。

    1. 友達にリンクを送りつける

    「おい、お前の悪口このブログに毎日書いてくから。よろしくな。」と告げ、URLを送る。友達がブログを読んでくれる。
